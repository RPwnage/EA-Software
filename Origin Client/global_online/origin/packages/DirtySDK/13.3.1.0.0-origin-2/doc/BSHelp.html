DirtySock BuildStudio Help

Tips on using BuildStudio for DirtySock  2007-07-03

Any combination of Platforms and Build Types can be chosen. BuildStudio will build each Built Type selected for each Platform selected in succession. Some of the Platforms have platform-specific Options that can be set. None of these options should be used currently. Please send any suggestions to Daniel Carter.


BuildStudio is set up to halt after a failed compilation. To continue building the rest of the Platform and Build Type combinations, go to the View drop down and choose Enable Try All Builds. This creates a new button, Try All Builds. Use this button to continue compilation of subsequent Platform and Build Type combinations after a build failure.

Tasks 

For each task, the BuildFile Location is needed. A default location is provided. Please make sure that this correctly identifies the DirtySDK.build file to be built.


Clean

  • Verbose - Provides the complete output.
  • Pass Config - When checked, this passes all the Platform Options in the Clean command. Currently, there are no Platform Options that need to be passed.

Build

  • Verbose - Provides the complete output.
  • Version - This Build Option is used to specify the release version to name the build, such as 5.5.0.
  • Date - This Build Option is used to specify the date of the build, such as "July 4, 2007". The quotes are necessary because of the spaces in the date.
  • Build sln - Not currently functioning. This Build Option refreshes the sln as a part of the build.
  • Build Targets - This Build Option is unused.
  • Xenon Destination Directory - This Build Option is Xenon-specific and should not be altered, unless it's effect is fully understood.
  • Note - As with all other build tools that have been used in the past, the build attempts to deploy to the default Xenon console. To specify the deploy targets, use the Deploy Task.

Deploy - Xenon-only

  • Verbose - Provides the complete output.
  • Xenon Target List - This Deploy Option allows the console that the build is to be deploy on to be specified. If nothing is present in the Target List, the build is deployed to the default console. To specify more than one target, enclose the list of consoles within quotes and separate them with spaces. Targets can be specified using either their IP Addresses or names.
  • Xenon Destination Directory - This Build Option is Xenon-specific and should not be altered, unless it's effect is fully understood.

Please contact Daniel Carter with any questions or suggestions.
