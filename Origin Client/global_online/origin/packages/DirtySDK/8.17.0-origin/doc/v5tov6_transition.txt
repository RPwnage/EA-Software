please write the "GOS dirtysock support" mailing list so those items can be added to future revisions of this
document.

API deprecations

    The following APIs and/or functionalities have been deprecated.  Client code referencing these APIs should
    be updated as follows:

    DirtySock (All)
        NetGameUtil
            Deprecated the ability for a single NetGameUtil ref to host multiple Comm refs.
            
            Deprecated bitfield macros (e.g. NETGAME_CONN_UDP) previously used to select protocol.  Instead,
            a CommAllConstructT function pointer is passed into NetGameUtilConnect().  This allows the caller
            to specify the protocol they want and not link in other protocols that are not wanted.

    DirtySock (DS)
        DirtyWmManager
            Deprecated DirtyWmManagerStatus() 'scbs' selector.

    DirtySock (Xenon)
        NetConnXenon
            Deprecated friends enumeration polling.
            
            Removed code that detected legacy usage of various NetConnStatus() control selectors with the
            addition of 0-3 to specify the gamertag and converted it to the current usage, where the gamertag
            index is specified in a separate variable instead.


API renaming or behavioral changes
                    
    The following APIs have been renamed, or the behavior has changed in a way that will require client code
    changes for any user code using the APIs.  User code referencing these APIs should be updated as follows:        

    DirtySock (All)
        LobbyNames
            LobbyNames from DirtySDK V5 has been deprecated and renamed to DirtyNames.  Please change any
            references accordingly for previous behavior.       

        LockerApi
            LockerApiFetchLocker2() has been deprecated, and LockerApiFetchLocker() has been changed to include
            the pGameName parameter.  Specify pGameName=NULL for previous behavior.
            
            LockerApiUpdateFileInfo2() has been deprecated, and LockerApiUpdateFileInfo() has been changed to
            include the iPerm parameter.  Specify iPerm=-1 for previous behavior.
            
        NetConn
            NetConnConnect2() has been deprecated, and NetConnConnect() has been changed to include the
            iData parameter.  Specify -Data=0 for previous behavior.

        NetGameDist
            NetGameDistSendStreamProc2() has been deprecated, and NetGameDistSendStreamProc() has been changed
            to include the subchan parameter.  Specify subchan=0 for previous behavior.
            
            NetGameDistRecvStreamProc2() has been deprecated, and NetGameDistRecvStreamPRoc() has been changed
            to include the subchan parameter.

            Call NetGameDistInputTweak(NetGameDistRefT *pRef, ‘lrcv’, 1); after setting up netgamedist, 
            starting the game, and most importantly after the loading screen. Before the first NetGameDistInputCheck
            is a simple efficient place.

        VoipTunnel
            Deprecated VOIPTUNNEL_MAXCLIENTS define.  Specify 128 for previous behavior.
            
            VoipTunnelCreate2() and VoipTunnelCreate3() have been deprecated, and VoipTunnelCreate() has been
            changed to include the iMaxClients and iMaxGames parameters.  Specify iMaxClients=128 and
            iMaxGames=1 for previous behavior.
            
        Voip
            VoipStartup2() has been deprecated, and VoipStartup() has been changed to include the iMaxLocal
            parameter.  iMaxLocal=1 should be specified for previous behavior.
            
            VoipControl2() has been deprecated, and VoipControl() has been changed to include the pValue
            parameter.  Specify pValue=NULL for previous behavior.
            
            VoipSetLocalUser() has been changed to include the bRegister parameter.  Specify bRegister=TRUE
            for previous behavior.
            
            VoipConnect2(), VoipConnect3(), and VoipConnect4() have been deprecated.  VoipConnect() has been
            changed to include the pUserId, uClientId, and uSessionId parameters.  Specify pUserId=NULL,
            uClientId=0, and uSessionId=0 for previous behavior.

    DirtySock (Xenon)
        NetGameUtil.h and ConnApi.h now depend on <xtl.h> and <xonline.h> being previously included.
        NetConnXenon
            Deprecated NetConnStatus() and NetConnControl() 'titl' selectors.  Specify 'xlsp' for previous
            behavior.
