Start a network transfer from a resource type and id value.

Parameters
*pState- module state
*pXfer- transfer descriptor
uType- type of resource to download
uIdent- identity of resource to download
*pCallback- function to call with update progress
iTimeout- number of milliseconds allowed for transaction
iPriority- NETRSRC_PRIO_* priority of item, if cache enabled (send 0 if don't care)
Returns
int32_t - Sequence number for the transaction, changes for each update Returns 0 if the request was for a cached entry and there's no sequence number to remember.
Note
The type/ident fields are internally translated into URLs via a mapping from the server and the file downloaded from a web server. The user supplied callback function handles buffer management and appending the data into the buffer. The timeout field lets the application specify a maximum time for the transfer.

Start a network transfer from a resource string.

Parameters
*pState- module state
*pXfer- transfer descriptor
*pResource- string for resource to download
*pCallback- function to call with update progress
iTimeout- number of milliseconds allowed for transaction
iPriority- NETRSRC_PRIO_* priority of item, if cache enabled (send 0 if don't care)
Returns
Result of NetReourceFetch - see return codes for that function
Note
This is an adapter for the NetResourceFetch which allows game code to request a resource in string format as opposed to type and ident. The string sent to this function should be in the format: "<4-character type>-<numeric ID value>" An optional prefix of "netres:" may be added if required, but will be ignored by NetResourceFetch. Example: "netres:imag-001"
