68  <b><a name="proto_dirtysock">Using a Dirtysock Protocol</a></b>
69 
70  Generally, there is a core set of functions a Dirtysock protocol implements:
71 
72  <ul>
73  <li>
74  <b>Proto<em>Protocol</em>Create</b> - handles the allocation of a private state structure
75  as well as basic initialization tasks. The reference pointer returned by the
76  Create function is used by the other protocol functions.
77 
78  Examples: ProtoHttpCreate(), ProtoPingCreate()
79  </li>
80 
81  <li>
82  <b>Proto<em>Protocol</em>Destroy</b> - releases the private state structure, terminates any
83  pending connections, and closes any open socket resources.
84 
85  Examples: ProtoSSLDestroy(), ProtoAriesDestroy()
86  </li>
87 
88  <li>
89  <b>Proto<em>Protocol</em>Request</b> - initiates a transaction request of some sort. The naming
90  will vary depending on the protocol in question.
91 
92  Examples: ProtoHttpGet(), ProtoPingRequest()
93  </li>
94 
95  <li>
96  <b>Proto<em>Protocol</em>Receive</b> - reads the response from a Request transaction. Returns
97  zero if no data is available. In general, the caller is responsible for handling
98  timeout errors.
99 
100  Examples: ProtoHttpInfo()/ProtoHttpData(), ProtoPingResponse(), ProtoSSLRecv()
101  </li>
102 
103  <li>
104  <b>Proto<em>Protocol</em>Update</b> - handles reading from the socket and any protocol parsing
105  involved, and generally stores the data within the protocol reference structure.
106  In some cases, Update() is called asynchronously via socket callback, in other
107  cases the user must periodically call the update function.
108 
109  Examples: _ProtoPingCallback() (private, called by socket callback) ProtoHttpUpdate() (public,
110  called by user).
111  </li>
112  </ul>
113 
114  Commonly, a single recurring state-based routine will handle these transactions. Example
115  psuedocode could look like this:
116 
117  <b><a name="proto_example">Example</a></b>
118 
119  \code
120 
121  int32_t MyRecurringNetFunc(MyNetRef *pMyNetState, char *pHostName, uint32_t timeout)
122  {
123  // if update function is not in a socket callback, call it here
124  if (pMyNetState->pRef != NULL)
125  {
126  ProtoUpdate(pMyNetState->pRef);
127  }
128 
129  switch(pMyNetState->eState)
130  {
131  // before we can do anything, we have to resolve the hostname
132  case MYPROTO_STATE_RESOLVE:
133  {
134  if (pMyNetState->pHost == NULL)
135  {
136  // request hostname resolve (async)
137  pMyNetState->pHost = SocketLookup(pHostName,timeout);
138  }
139  else
140  {
141  int32_t host;
142 
143  // check to see if the hostname is resolved yet
144  host = pMyNetState->pHost->Done(pMyNetState->pHost);
145  if (host == -1)
146  {
147  // we've timed out - move to destroy state
148  pMyNetState->eState = MYPROTO_STATE_DESTROY;
149  }
150  else if (host != 0)
151  {
152  // init the protocol, and move to request state
153  pMyNetState->pRef = ProtoCreate();
154  pMyNetState->eState = MYPROTO_STATE_REQUEST;
155  }
156  }
157  }
158  break;
159 
160  case MYPROTO_STATE_REQUEST:
161  {
162  // make the transaction request
163  ProtoRequest(pMyNetState->pRef,...);
164 
165  // advance to receive state
166  pMyNetState->eState = MYPROTO_STATE_RECEIVE;
167  }
168  break;
169 
170  case MYPROTO_STATE_RECEIVE:
171  {
172  if (ProtoReceive(pMyNetState->pRef,...))
173  {
174  // we've got some data... do something with it
175  _MyNetReadDataFunc(pMyNetState);
176 
177  // done - advance to destroy stage
178  pMyNetState->eState = MYPROTO_STATE_DESTROY;
179  }
180  }
181  break;
182 
183  case MYPROTO_STATE_DESTROY:
184  {
185  if (pMyNetState->pHost)
186  {
187  // release hostname resolve socket
188  pMyNetState->pHost->Free(pCmdRef->pHost);
189  pMyNetState->pHost = NULL;
190  }
191 
192  if (pMyNetState->pRef)
193  {
194  // release protocol
195  ProtoDestroy(pCmdRef->pRef);
196  pMyNetState->pRef = NULL;
197  }
198 
199  pMyNetState->eState = MYPROTO_STATE_INACTIVE;
200  }
201  break;
202 
203  case MYPROTO_STATE_INACTIVE:
204  {
205  }
206  break;
207  }
208  }
209 
210  \endcode
211 
212  In many cases, some of these states will be broken out into separate functions, when
213  multiple transactions are required.
