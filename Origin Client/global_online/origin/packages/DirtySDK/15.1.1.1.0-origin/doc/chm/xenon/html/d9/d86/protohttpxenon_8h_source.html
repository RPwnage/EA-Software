2 /*!
3  \File protohttpxenon.h
4 
5  \Description
6  Implements the ProtoHttp API, allowing use of either the generic ProtoHttp
7  code used on other platforms or the XHTTP API that is Xenon-specific.
8  Determination of which API to use can be made at run-time on a per-instance
9  basis. The default API is the generic ProtoHttp implementation; the 'xhtp'
10  control selector is used to switch between implementations.
11 
12  \Notes
13  The goal of protohttpxenon is to accomplish the following set of requirements:
14  - Wrap the Xenon-specific XHTTP API in the ProtoHttp interface.
15  - Continue to allow use of the generic ProtoHttp interface.
16  - Have XHTTP/ProtoHttp use selectable at run-time on a per-instance basis.
17  - Not require modification of the generic ProtoHttp code to accomplish this.
18 
19  To accomplish this goal, a somewhat complicated solution was implemented. The
20  protohttpxenon.c file includes protohttpxenon.h (and protohttp) twice. The first
21  pass defines an alternate namespace for the 'generic' ProtoHttp functions, by
22  using preprocessor defines to map ProtoHttpSomething to __ProtoHttpSomething.
23  When protohttp is subsequently included, the ProtoHttp public interface is
24  remapped to an underscore-prefixed version.
25 
26  The second pass undefines these mappings and re-includes protohttp.h. This
27  second pass restores the public API as owned by the protohttpxenon implementation
28  of the public API.
29 
30  The end result is that calls to ProtoHttp-namespace functions are handled
31  by protohttpxenon, but protohttpxenon is able to access the generic API by
32  using the underscore-prefixed names. It is therefore able to select at
33  run-time, on a per-instance basis, which version of the API it is going
34  to use, without needing to modify the generic code.
35

38 
39  \Version 1.0 01/23/2012 (jbrookes) First Version
40 */
