25  \Description
26  The Dirtysock module provides a portable sockets implementation.
27 
28  <b>Overview</b>
29 
30  Dirtysock is a platform independent API providing access to common network library
31  functions. It it similar in many respects to BSD/Sockets and Winsock, but targeted
32  specifically for a real-time game networking environment. It will be helpful to have
33  an understanding of sockets before reading this documentation.
34 
35  Unlike BSD and Windows sockets, only the AF_INET address family is supported, and blocking
36  read and write calls are not supported. Instead, the Dirtysock API supports two
37  extensions not present in a normal socket environment that allow user-defined callbacks
38  and idle functions. The SocketCallback() function allows you to register a function
39  that gets called when an incoming packet arrives on a particular socket. This can be
40  used to process incoming packets in essentially real-time. If the \em idle parameter
41  of SocketCallback() is nonzero, the registered function will be called approximately
42  every \em idle milliseconds. This can be used to deal with resends and things of
43  that nature. Callback and idle functions are called by a private Dirtysock thread.
44

90  <b>Examples</b>
91 
92  <em>Create a new stream socket connection</em>
93 
94  \code
95 
96  struct sockaddr peeraddr;
97 
98  pSock = SocketOpen(AF_INET, SOCK_STREAM, 0);
99  if (pSock != NULL)
100  {
101  // init sockaddr structure
102  SockaddrInit(&peeraddr, AF_INET);
103  SockaddrInSetAddr(&peeraddr, iAddr);
104  SockaddrInSetPort(&peeraddr, iPort);
105 
106  // start the connect
107  SocketConnect(pSock, (struct sockaddr *)&peeraddr, sizeof(peeraddr));
108  }
109 
110  \endcode
111 
112  <em>Check for connection ready</em>
113 
114  \code
115  // note: as connection resolution is asynchronous, SocketInfo() is usually
116  // called repeatedly until either the connection is ready or a given timeout occurs
117  if (SocketInfo(pSock, 'stat', NULL, 0) > 0)
118  {
119  NetPrint("Connection ready\n");
120  }
121 
122  \endcode
