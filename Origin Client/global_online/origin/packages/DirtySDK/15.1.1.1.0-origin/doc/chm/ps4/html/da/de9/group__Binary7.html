Decode binary data from 7-bit high-ASCII characters to 8-bit data.

Parameters
*pDst- destination buffer (will hold decoded data)
iDstLen- destination buffer max size (in bytes)
*pSrc`- source buffer
Returns
int32_t - < 0 indicates error; >= 0 indicates bytes in converted data
Note
Decoding will stop at the first low-ASCII character, or until dlen bytes have been written, whichever comes first. Storage requirements are slen*7/8 bytes.

Encode binary data to fit in 7-bit high-ASCII characters. All of the requested data will be encoded in a reversible form that is slightly larger than the original data.

Parameters
*pDst- destination buffer (will hold encoded data)
iDstLen- destination buffer max size (in bytes)
*pSrc- source buffer
iSrcLen- source buffer length (in bytes)
bTerminate- TRUE to terminate the buffer, else FALSE.
Returns
int32_t - < 0 indicates error; >= 0 indicates bytes in converted data
Note
If there is insufficient space to store all the encoded data, an error is returned immediately; no data is encoded or written. Storage requirements are (slen*8+6)/7 bytes.

01/20/2004 (djones)
