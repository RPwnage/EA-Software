156  * Note1: this is an optional callback which is called after new data has been
157  * received and buffered or at other times where the protocol state has
158  * significantly changed. It is called by a private thread so the routines it
159  * uses must be thread safe. It can be used to provide "life" to servers or
160  * other modules which use comm services. The code must not take too long to
161  * execute since it will impact comm performance if it does.
162  *
163  * Note2: By disabling and enabling this callback at specific times, it is
