DIRTYCODE_API int32_t NetGameDistControl ( NetGameDistRefT *  pRef,
int32_t  iSelect,
int32_t  iValue,
void *  pValue 
)

Set NetGameDist operation parameters.

Parameters
*pRef- reference pointer
iSelect- item to tweak
iValue- tweak value
pValue- pointer tweak value
Returns
int32_t - selector specific
Note
select can be one of the following:

-— basic setup and control -—
'clri' - clear the local queue of inputs
'mini' - min window size clamp
'maxi' - max window size clamp
'spam' - sets debug output verbosity (0...n)

— advanced setup
'crcs' - enable/disable the reception of CRC challenges
'lcrc' - to provide the current CRC as request by NetGameDist
'lrcv' - local recv flow control. Set whether we are ready to receive or not.
'lsnd' - local send flow control. Set whether we are ready to send or not.

-— provide information -—
'late' - overall latency of game
'plat' - get the end-to-end latency in number of packets
'rate' - current exchange rate
'rcrc' - get the remote CRC. if iValue==0, returns whether we have a value. Otherwise return the value itself.
'rrcv' - remote recv flow control. Tells whether remote is ready to receive or not.
'rsnd' - remote send flow control. Tells whether remote is ready to send or not.
'wind' - input exchange window
'?cmp' - offset of first packet to process in input buffer
'?cws' - can we send. FALSE if sending would overflow the send queue, iValue is length
'?inp' - offset of last packet to process in input buffer
'?out' - offset of last packet to send in output buffer
'?snd' - offset of first packet to send in output buffer

The count of [sndoff..outoff] is the number of packets ready to send. The min of [cmpoff..inpoff] and [cmpoff..outoff] is the number of packets ready to process (because you need a packet from each peer in order to perform processing).
Version
1.1 11/18/08 (jrainy) refactored from NetGameDistInputTweak
