Detailed Description

Simple stream cipher version 2 (SSC2) designed to provide reasonable data encryption. This algorithm was designed from scratch by GWS and is based on the general methods I have seen used in other stream ciphers. As a result, it has obviously not been cryptoanalyzed by anyone so problems may exist (and in fact I found a major data leak in version 1). However, for data obscuring and things of that nature, this should work fine. Since it was designed from scratch it should be free from intellectual property issues.

In order to use a stream cipher like this, the initial key sequence used to initialize it must be unique for every usage. This is typically done by appending on some random bytes (which are also appended on by the decoder). The fact that the random bytes are transmitted as cleartext does not matter. If the key is not made unique with random bytes, then the data becomes relatively easy to decode (at least by crypto standards).
