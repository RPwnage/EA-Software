Detailed Description

Provide a wrapper that translates the Winsock network interface into DirtySock calls, as well as handling Xenon-specific functionality. In the case of Winsock, little translation is needed since it is based off BSD sockets (as is DirtySock).

Register a callback routine for notification of socket events. Also includes timeout support.

Parameters
*pSocket- socket reference
mask- valid callback events (CALLB_NONE, CALLB_SEND, CALLB_RECV)
idle- if nonzero, specifies the number of ticks between idle calls
*ref- user data to be passed to proc
*proc- user callback
Returns
int32_t - zero
Note
A callback will reset the idle timer, so when specifying a callback and an idle processing time, the idle processing time represents the maximum elapsed time between calls.
