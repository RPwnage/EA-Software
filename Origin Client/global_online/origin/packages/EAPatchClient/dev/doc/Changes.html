EAPatchClient Changes

Version 1.05.02 (Unpublished)

Release notes:

  • This release primarily fixes a bug related to file buffering that would cause written files to be mis-written.

Changes

  • Fixed a bug in ErrorBase::HandleError where it wouldn't set mbSuccess to false like it should.
  • Fixed a bug in the File class where it wasn't AddRef-ing its mStreamBuffer member, which could have resulted in a crash.
  • Fixed File::GetStream to return &mStreamBuffer instead of mpStream. This was the cause of the mis-written file bug noted above in the release notes.
  • Added the FileComparer class, which is primarily of internal interest and not typically of interest to users of this package.
  • Fixed OpenTwoFiles to return the correct result for the case that the first file opens successfully but the second doesn't. This is likely only of internal interest.

Version 1.05.01 (March 4, 2013)

  • Fixed a bug in the HTTP module with respect to maintaining DownloadStats::mExpectedBytes.
  • Revised all Reset functions to guarantee that all memory is freed.
  • Added support for the most recent versions of DirtySDK, which implemented an API change.

Version 1.05.00 (January 11, 2012)

  • Implemented a fix for a PS3 EAIO build problem related to EAIO version identification.
  • Added initial test code for writing to PS3 and XBox 360 hard drives.
  • Updated unit tests to support more platforms (e.g. clang/osx).

Version 1.04.00 (December 4, 2012)

  • Added HTTP status reporting.
  • Fixed some logic bugs.
  • Improved status callback support.
  • Simplified the requirements for the .eaPatchInfo and .eaPatchImpl files.
  • Added additional unit tests.

Version 1.03.00 (Novmber 1, 2012)

  • Added telemetry support.
  • Added status callback support.
  • Added runtime metrics support.
  • Improved the unit tests, including allowing usage of an out-of-process HTTP server.
  • Added platform support for Unix, Mac, next gen consoles, and (theoretically, untested) mobile.

Version 1.02.00 (October 1, 2012)

  • Improved error handling reporting.
  • Implemented EAPatchBuilder, plus some initial data-driven unit tests.
  • Implemented EAPatchHTTP as a publicly accessible class.
  • Added some additional flexibility to EAPatchDirectory.

Version 1.01.00 (September 9, 2012)

  • Added error handling.
  • Implemented EAPatchDirectory
  • Fixed logic bugs.
  • Added initial unit tests.

Version 1.00.00 (August 2, 2012)

This is the first published version of EAPatchClient. It includes minimal initial support for PS3, XBox 360, and PS3.
