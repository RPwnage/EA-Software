EAPatchMaker Changes

Version 1.06.00 (Unpublished)

  • Fixed a crash bug. If you were previously getting a crash then this probably fixes it.
  • Added the -blockSize command line argument. Allows you to force usage of a given block size. Ignored if -optimize is enabled.

Version 1.05.01 (April 15, 2013)

  • Made it so that all Reset functions recursively free memory.

Version 1.04.00 (December 4, 2012)

  • Made it so that a generated .eaPatchImpl file can be recycled as an input to EAPatchMaker. Previously such an input to EAPatchMaker would result in its old patch files being persisted into the new .eaPatchImpl file, which is surely not what you want.

Version 1.03.00 (Novmber 1, 2012)

  • Added additional command line options.
  • Added the ability to recognize relative URLs.

Version 1.02.00 (October 1, 2012)

  • Improved error detection reporting.
  • Relaxed the requirements for a minimal .eaPatchImpl file.

Version 1.01.00 (September 9, 2012)

  • Added additional command line options.
  • Improved error and status output.

Version 1.00.00 (August 2, 2012)

This is the first published version of EAPatchMaker.
