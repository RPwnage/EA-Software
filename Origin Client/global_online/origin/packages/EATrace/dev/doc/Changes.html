EATrace Changes

Changes

Version 2.09.02 (July 3rd, 2012)

Changes

  • Fixed an Android missing #include for the case of non-bulk-builds.
  • Made it so mobile EAOutputDebugString prints to stdout instead of stderr, for consistency with other platforms.
  • Added additional platform support.

Version 2.09.01 (May 7, 2012)

Changes

  • Changed ZONE_OBJECT_MIN_ALIGN to unilaterally be 16. This has no effect on 64 bit platforms but increases by 8 bytes for 32 bit platforms. While more memory is being used, it's probably small compared to the total being allocated. This change allows ZoneObject to work with heaps other than PPMalloc and allows it to work with 32 bit PPMalloc when PPM_MIN_ALIGN is set to 16.
  • Updates for better handling of Microsoft API partitions.
  • Adding appropriate wchar_t function overloads for compilers that treat char16_t/char32_t and wchar_t as distinct types.
  • Added support for the clang compiler under unix64 configurations.

Version 2.09.00 (March 6, 2012)

Changes

  • Added PSVita platform support (EA_PLATFORM_PSP2).
  • Added fix for signed/unsigned mismatch warnings on PC with char and wchar_t.
  • Made it so the trace macros don't read EA::Trace::TraceHelper::isTracingEnabled directly but use EA::Trace::TraceHelper::GetTracingEnabled() instead. This makes for more predictable behavior in the presence of DLL usage.
  • Incorporated usage of __analysis_assume in assertion code, to assist with Microsoft /analysis testing.
  • Made EALogConfig enabled by default. Doing so required a few changes to EALogConfig.cpp to allow compiling, but the functionality is unchanged.
  • Fixed an EDG compiler warning related to type aliasing in RefCount.h.

Version 2.08.02 (October 19, 2011)

Changes

  • Added support for the Cafe (Wii U) platform.
  • Revised the definition of EA_DEBUG_BREAK for Apple platforms to be better.
  • Added kLevelPrivate. Refers to a private (local-author-only) debug level, expected to be off by default.
  • Added GetAllocator/SetAllocator, which allows EATrace to have a package-specific ICoreAllocator. By default SetAllocator returns the ICoreAllocator default allocator, but the user can override it by calling EA::Trace::SetAllocator.
  • Added Get/SetTracingEnabled to TraceHelper, with a corresponding TraceHelper::sTracingEnabled field that is used to gate the EA_TRACE_HELPER_MESSAGE_GL macros. Implemented by Andrew Willmott.
  • Added SetAllEnabled(bool enabled) to IServer and TraceHelperTable. This is useful for resetting all user-disabled asserts. Implemented by Andrew Willmott.

Version 2.08.01 (August 31, 2011)

Changes

  • Fixed a bug introduced in v2.08.00 which casued EATRACE_GLOBAL_ENABLED to be defined as 0 by default instead of 1, which it's supposed to be.
  • Revised the unit tests to use EATest as a test framework.
  • Added EA_PLATFORM_CTR (i.e. Nintendo 3DS), iPhone, and Palm platform support for EA_DEBUG_BREAK.
  • Fixed a possible deadlock that results from an AutoMutex in Server::Trace.

Version 2.08.00 (July 5, 2011)

Changes

  • Added virtual void SetFlushOnWrite/GetFlushOnWrite to ILogReporter and ITracer, which allows users to specify on a log-by-log basis whether log writes are flushed immediately after they occur. Also added the EATRACE_FLUSH_ON_WRITE_ENABLED_BY_DEFAULT config option to specify the default initial value for SetFlushOnWrite.
  • Added TraceHelper::TraceFormattedV(va_list) to complement TraceHelper::TraceFormatted(...)

Bug Fixes
  • <none>

Version 2.07.02 (March 14, 2011)

Additions

  • Added GetDefaultDisplayTraceDialogFunc().
  • Added the EATRACE_DEBUG_BREAK_ABORT_ENABLED config parameter, which lets the user specify that abort is used instead of debug-break.
  • Added kAlertResultAbort and made EA_ASSERT on Android use it, since debug breaking isn't supported on Android.
Bug Fixes
  • Fixed a couple errors related to the definition and usage of EA_TRACE_ENABLED, EA_LOG_ENABLED, and EA_TRACE_REPORTING_LOCATION. Reported by Jeffrey Sturgeon.
Changes
  • Added some default group names to TraceHelper::TraceHelper. Suggested by Ionut Matasaru.
  • Added the EA_TRACE_CURRENT_FUNCTION_LEVEL config option, which specifies how detailed __FUNCTION__ output is.
  • Added the EATRACE_GLOBAL_ENABLED option. If enabled then the Trace system is shared between the application and its DLLs. If disabled then each compilation unit gets its own copy of EATrace.
  • Revised the Release function to set the ref count to 0 for debugging purposes.
  • Made it so that EA_ASSERT and EA_VERIFY append \n to expression strings. This is useful for the case that assertions are logged to disk. Suggested by Ionut Matasaru.
  • Made it so that dialog boxes by default appear on desktop platforms, but not on console platforms. Previously they would appear by default on XBox 360, which probably is not what most users want.
  • If dialogs boxes are disabled then assertions cause debug breaks, whereas previously they would do nothing.
  • Made it so that EA_TRACE_CA_NEW and EA_TRACE_CA_NEW_ARRAY don't pass on debug parameters (e.g. allocation name) in release builds.

Version 2.07.01 (November 5, 2010)

Additions

  • Added a print EXIT(n) at the end of the unit tests on wii to support automated running/testing on Wii.

  • Added sample LogReporter code to the unit test.

Version 2.06.02 (November 24, 2008)

Release Notes

  • This release removes a dependency on the UTFFoundation package. By doing so it requires that the EAStdC version used with it is v1.03.08 or later.
  • The Framework2/nant build of this package doesn't build EALogConfig.cpp any more. The reason for this is that EALogConfig.cpp is still depenent on UTFFoundation, and this package otherwise removes use of UTFFoundation. If you need EALogConfig.cpp in your Framework2-based build, please notify Paul Pedriana and we can get it working sooner rather than later.

Version 2.06.01 (March 20, 2008)

Additions

  • Added documentation clarifying that you cannot use dynamically calculated EA_LOG group names and how you can work around it.

Version 2.05.00 (May 4, 2007)

Release Notes

  • This version makes EATrace transparently compatible with the EAAssert package. See the notes below in the Changes section.

Additions

  • <none>
Bug Fixes
  • EA::Trace::Configure now longer makes a redundant call to Server::Init().

  • Made EATrace explicitly recognize the EAAssert package and be able to use EAAssert's functionality if configured to do so. Recall that EAAsset is a package that is a subset of EATrace which covers most of the assertion functionality present in EATrace. We want the to packages to co-exist cleanly.
  • Moved unit tests from UTFFoundation to EATrace.
  • Some slight cleanup to EALogConfig ConfigureReporter(). Submitted by Jon Parise.
  • Made EATrace explicitly recognize the EAAssert package and be able to use EAAssert's functionality if configured to do so. You don't need to have EAAssert present and EATrace won't see it unless configured so. Recall that EAAsset is a package that is a subset of EATrace which covers most of the assertion functionality present in EATrace.
